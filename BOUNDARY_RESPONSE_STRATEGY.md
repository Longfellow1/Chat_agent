## 设计原则

### 1. 分层检测 (Layered Detection)

按优先级顺序检测6个类别，第一个匹配即返回：

```
金融预测 (Financial Prediction)
    ↓
实时监控 (Impossible Services)
    ↓
医疗诊断 (Medical Diagnosis)
    ↓
法律文件 (Legal Documents)
    ↓
对抗性表述 (Adversarial Requests)
    ↓
商业计划 (Business Plans)
```

### 2. 置信度评分 (Confidence Scoring)

每个模式都有置信度 (0.0-1.0)：
- 1.0: 直接模式，明确违反能力边界
- 0.9: 强模式，高度可能违反能力边界
- 0.8: 中等模式，可能违反能力边界
- 0.7: 弱模式，需要谨慎判断

**阈值**: 置信度 >= 0.7 才触发拒识

### 3. 场景化回复 (Scenario-Specific Replies)

每个类别有专门的回复模板，而不是通用拒识：

```python
金融预测 → "我无法预测股市走势或保证投资收益..."
医疗诊断 → "我无法提供医疗诊断或开具处方..."
法律文件 → "我无法生成具有法律效力的合同..."
实时监控 → "我目前无法提供实时监控或定时推送服务..."
```

## 6个检测类别详解

### 类别1: 金融预测 (Financial Prediction)

**触发条件**: 绝对化承诺 + 金融相关

**模式示例**:
```
"一定涨停" / "必定涨" / "保证盈利" / "100%赚钱"
"哪只股票一定涨停"
"直接告诉我下周A股哪只股票一定涨停"
```

**回复**:
```
我无法预测股市走势或保证投资收益。
金融市场存在不确定性，任何投资都有风险。
建议您根据自身风险承受能力，咨询专业投资顾问后谨慎决策。
```

**测试用例**: ✅ 3/3 通过

---

### 类别2: 实时监控 (Impossible Services)

**触发条件**: 持续性服务 + 不可能的频率

**模式示例**:
```
"实时监控" / "持续跟踪" / "24小时监控"
"每分钟推送" / "每秒推送" / "每小时更新"
"请帮我实时监控全球突发新闻并每分钟推送"
```

**回复**:
```
我目前无法提供实时监控或定时推送服务。
我可以帮您查询当前的信息，但无法持续跟踪或自动推送。
如需实时监控，建议使用专业的监控工具或订阅相关服务。
```

**测试用例**: ✅ 3/3 通过

---

### 类别3: 医疗诊断 (Medical Diagnosis)

**触发条件**: 医疗行为 + 直接/确定性表述

**模式示例**:
```
"直接判断" / "直接诊断" / "确诊"
"给处方" / "开药方"
"我有胸痛三天，帮我直接判断是不是心梗并给处方"
```

**回复**:
```
我无法提供医疗诊断或开具处方。
您描述的症状可能需要专业医生评估。
建议您尽快就医或拨打急救电话，以便获得及时准确的诊疗。
```

**测试用例**: ✅ 3/3 通过

---

### 类别4: 法律文件 (Legal Documents)

**触发条件**: 法律文件 + 直接生成/通用

**模式示例**:
```
"直接生成合同" / "可直接签署"
"国际合同" / "法律意见"
"适用于所有国家的签证法律意见"
"帮我直接生成一份可直接签署的国际合同"
```

**回复**:
```
我无法生成具有法律效力的合同或提供法律意见。
法律文件需要根据具体情况定制，并由专业律师审核。
建议您咨询专业律师，确保文件的合法性和有效性。
```

**测试用例**: ✅ 3/3 通过

---

### 类别5: 对抗性表述 (Adversarial Requests)

**触发条件**: 明确对抗 + 强制要求

**模式示例**:
```
"不要建议我咨询专业人士"
"我就要一个确定答案"
"不要废话"
"直接给我答案"
```

**回复**:
```
我理解您希望快速获得答案，但某些问题确实超出了我的能力范围。
为了您的利益，我建议咨询相关领域的专业人士。
我可以提供一般性信息，但无法替代专业意见。
```

**测试用例**: ✅ 2/2 通过

---

### 类别6: 商业计划 (Business Plans)

**触发条件**: 商业相关 + 保证成功

**模式示例**:
```
"100%保证盈利的商业计划"
"一定成功的创业方案"
"可执行且保证盈利"
```

**回复**:
```
我无法提供保证盈利的商业计划。
创业存在不确定性，成功受多种因素影响。
我可以提供一般性的创业思路，但建议您结合市场调研和专业咨询制定具体方案。
```

**测试用例**: ✅ 2/2 通过

### 文件结构

```

tests/unit/
└── test_boundary_response.py     # 新增：23个单元测试
```

### 核心函数

```python
# 检测边界请求
def detect_boundary_response(query: str) -> Optional[BoundaryMatch]:
    """返回 BoundaryMatch 对象或 None"""
    
# 获取场景化回复
def get_boundary_reply(query: str) -> Optional[str]:
    """返回回复文本或 None"""
    
# 数据结构
@dataclass
class BoundaryMatch:
    matched: bool           # 是否匹配
    category: str          # 类别
    confidence: float      # 置信度 0.0-1.0
    reply_template: str    # 回复模板
```

### 集成到 chat_flow.py

```python
# 导入
from app.policies.boundary_response import get_boundary_reply

# 在 run() 函数中使用
if detect_boundary_response(query):
    boundary_reply = get_boundary_reply(query)
    resp = ChatResponse(
        ...
        final_text=boundary_reply or "默认回复",
    )
    return resp
```
