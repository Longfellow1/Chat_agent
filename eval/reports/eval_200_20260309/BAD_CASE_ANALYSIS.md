## 三、核心问题分析

### 问题1: search_nearby工具准确率极低 (7.1%)

**影响范围**: 28条样本，26条失败  
**核心原因**: 缺少GPS城市信息导致missing_slots

#### 典型Bad Case

```
H00001: "帮我搜一下附近有没有好吃的日料"
- 期望: search_nearby → tool_call
- 实际: find_nearby → clarify (missing_slots: city)
- 问题: 车载场景应自动获取GPS城市，但当前实现要求用户补充city参数
```

#### 失败模式

1. **missing_slots (26条)**: 所有"附近"查询都因缺少city参数而要求澄清
   - H00001-H00003, H00013, H00023-H00025, H00034-H00037, H00057, H00059-H00061, H00064-H00065, H00067, H00071, H00075, H00086, H00089, H00096-H00098

2. **wrong_mode (2条)**: 误判为reply而非tool_call
   - H00058: "帮我找个可以停车充电的地方" → reply
   - H00072: "这边哪里能存钱" → reply

#### 优化建议

**P0 - 紧急修复**:

1. **添加GPS城市自动填充逻辑**
   ```python
   # 在location_intent.py中添加
   def extract_location_params(query: str, gps_city: str = None) -> dict:
       params = {...}
       # 如果有"附近"/"周边"等词且无city参数，使用GPS城市
       if has_nearby_keyword(query) and not params.get("city"):
           params["city"] = gps_city or "当前位置"
       return params
   ```

2. **修改API接口支持GPS参数**
   ```python
   # server.py
   class ChatRequestBody(BaseModel):
       query: str
       gps_city: Optional[str] = None  # 新增GPS城市参数
   ```

3. **预期提升**: 26条 → 准确率从7.1%提升至93%+

### 问题2: 拒识能力完全缺失 (0.0%)

**影响范围**: 11条样本，全部失败  
**核心原因**: 无reject决策模式，所有超出能力边界的请求都被误判为reply或tool_call

#### 典型Bad Case

```
H00019: "帮我设个明天早上8点的闹钟"
- 期望: reject (超出能力边界)
- 实际: reply (给出如何手动设置闹钟的建议)
- 问题: 应该明确拒绝，而非提供变通方案

H00027: "比亚迪股票值得买吗"
- 期望: reject (投资建议超出边界)
- 实际: get_stock → tool_call (查询股价)
- 问题: 应该拒绝投资建议，而非查询股价
```

#### 失败模式

1. **reply误判 (7条)**: 超出能力边界但给出建议
   - H00019: 设闹钟 → 教用户手动设置
   - H00020: 发微信 → 提供文案建议
   - H00021: 打开空调 → 说明无法控制设备
   - H00022: 导航回家 → 要求提供地址
   - H00068: 订酒店 → 说明无法预订
   - H00094: 适合买房吗 → 给出建议

2. **tool_call误判 (4条)**: 超出能力边界但调用工具
   - H00027: 股票值得买吗 → get_stock
   - H00028: 应该买茅台还是五粮液 → get_stock
   - H00069: 买高铁票 → plan_trip
   - H00070: 查高铁余票 → plan_trip
   - H00095: 茅台值不值得长期持有 → get_stock

#### 优化建议

**P0 - 紧急修复**:

1. **添加reject决策模式**
   ```python
   # router.py中添加拒识规则
   REJECT_PATTERNS = [
       r"(设|定)(闹钟|提醒)",
       r"(发|转发)(微信|短信|消息)",
       r"(打开|关闭|调节)(空调|音乐|车窗)",
       r"导航(到|去|回)",
       r"(订|预订)(酒店|机票|车票)",
       r"(买|购买)(票|券)",
       r"(值得|应该|建议)(买|投资|持有)",
       r"(适合|该不该)(买房|投资)",
   ]
   ```

2. **修改intent_probs添加reject类别**
   ```python
   # 当前只有1-7类，需要添加类别8: reject
   intent_probs = {
       "1": 0.02,  # get_weather
       "2": 0.03,  # get_news
       ...
       "8": 0.85,  # reject (新增)
   }
   ```

3. **预期提升**: 11条 → 准确率从0%提升至90%+
