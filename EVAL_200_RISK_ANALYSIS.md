**样例**:

- "帮我发条微信给我老婆" (期望reject)

- "帮我订酒店" (期望reject - 无订票能力)

**风险评估**:

### 🟡 风险3: 难例 - 工具混淆 (102条)

**问题**: 88条工具调用全部标注为"难例"，存在工具混淆风险


**3.1 search_nearby vs web_search (最高风险)**

**混淆样例**:
- "帮我搜一下附近有没有好吃的日料" → 期望search_nearby
  - 风险: "搜"字可能触发web_search
- "搜一下周边有什么停车场" → 期望search_nearby
  - 风险: "搜"字强信号
- "帮我找找成都有哪些火锅店" → 期望web_search
  - 风险: 可能误判为search_nearby

**关键区分**:
- 有"附近/周边/这里"词 → search_nearby
- 无位置词但有城市名 → web_search
- 当前路由器可能在"搜"字上误判

**预期准确率**: 70-80%

**3.2 plan_trip vs web_search**

**混淆样例**:
- "帮我搜一下从北京去上海的攻略" → 期望web_search
  - 风险: "攻略"可能触发plan_trip
- "上海2日游攻略帮我搜一下" → 期望plan_trip
  - 风险: "搜"字可能触发web_search

**关键区分**:
- "X日游" + "攻略" → plan_trip
- 只有"攻略"无天数 → web_search
- "交通攻略" → web_search

**预期准确率**: 80-85%

**3.3 双意图查询**

**样例**:
- "帮我查下深圳今天的新闻和天气" → 期望get_news
- "特斯拉今天股价多少最近有什么新闻" → 期望get_stock

```python
]
```

**2. 优化search_nearby vs web_search**
```python
# 强化"附近/周边"信号
```
