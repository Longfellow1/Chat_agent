# 提示词优化 - 排除法修复总结

## 问题发现

在验证新提示词效果时，发现"我想去北京3天"被误判为纯闲聊（tool=None），而不是plan_trip。

## 根本原因

排除法检测的关键词列表不完整。排除法的逻辑是：
```python
if not any(kw in query for kw in tool_keywords):
    return None  # 纯闲聊
```

但原来的`tool_keywords`列表中缺少：
- 城市名（北京、上海等）
- 类别关键词（餐厅、酒店等）
- 出行相关词（出去、出游、度假等）

导致"我想去北京3天"不包含任何`tool_keywords`中的词，被误判为纯闲聊。

## 修复方案

扩展`tool_keywords`列表，包含所有可能触发工具的词汇：

```python
tool_keywords = [
    # plan_trip - 旅游/出行意图
    "旅游", "旅行", "游玩", "攻略", "行程", "去哪", "目的地", "出去", "出游", "度假",
    # find_nearby - 位置查询
    "附近", "周边", "这里", "附近有", "哪里有", "最近的", "周围",
    # get_weather - 天气查询
    "天气", "温度", "下雨", "晴天", "气温", "风", "穿衣指数", "冷", "热", "带伞",
    # web_search - 搜索行为
    "搜", "查", "找", "看看", "了解", "官网", "搜索", "查询",
    # get_news - 新闻
    "新闻", "消息", "资讯", "头条", "大事", "发生了什么",
    # get_stock - 股票
    "股票", "股价", "涨", "跌", "行情", "A股", "沪深",
    # 目的地（城市名）
    "北京", "上海", "广州", "深圳", "杭州", "西安", "成都", "南京", "苏州", "武汉",
    # 类别关键词
    "餐厅", "酒店", "景点", "医院", "银行", "超市", "加油站", "停车场", "火锅", "小吃",
]
```

## 修改文件

  - `RuleBasedRouter.try_route()` - 扩展排除法检测的关键词列表
  - `Router4BWithLogprobs.route()` - 同步修复
