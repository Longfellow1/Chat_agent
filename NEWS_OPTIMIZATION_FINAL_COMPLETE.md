# 新闻工具优化最终完成

## 问题回顾

用户反馈3个核心问题：
1. **重写提示词太精炼**：没有口语化，没人味儿
2. **相关性排序不明确**：不知道最相关的内容是否在最前面
3. **重复搜索无去重**：连续搜索同一query返回重复内容

## 解决方案

### 1. 重写提示词优化（口语化）

**修改文件**：`mcp_gateway.py` - `_apply_news_rewrite()`

**原提示词**：
```
你是一个专业的新闻编辑。你的任务是将新闻列表重写为简洁、清晰的摘要。
要求：每条新闻用一句话表达，不超过50字...
```

**新提示词**：
```
你是一个有趣的新闻编辑，擅长用轻松、口语化的方式讲故事。

要求：
1. 用口语化的方式重写每条新闻，像跟朋友聊天一样自然
2. 保留关键信息，但要生动有趣，不要太正式
3. 可以用"据说"、"看来"、"竟然"等口语词汇
4. 每条新闻控制在30-50字，简洁但有温度
```

**示例输出**：
```
1. 春节档电影火爆
今年春节档真是热闹，各种题材的电影都有，票房创新高，看来大家过年都爱去电影院。

2. 明星恋情曝光
某女星疑似有新恋情了，爆料的人说两人已经低调交往好几年，吃瓜群众表示很期待官宣。
```

### 2. 相关性排序优化

**修改文件**：`baidu_web_search_provider.py`

**改动**：提高相关性阈值，过滤不相关结果

```python
# 原：relevance_threshold=0.1（太低，不相关结果也被保留）
# 新：relevance_threshold=0.3（提高阈值，过滤不相关结果）
```

**排序逻辑**（已存在于`search_result_processor.py`）：
```python
# 综合评分 = 相关性 * 0.7 + 可信度 * 0.3
result["score"] = result["relevance"] * 0.7 + credibility_normalized * 0.3
unique.sort(key=lambda x: x.get("score", 0), reverse=True)
```

**效果**：
- ✅ 最相关的内容排在最前面
- ✅ 不相关的结果被过滤掉
- ✅ 综合考虑相关性和可信度

### 3. 重复搜索去重

**修改文件**：`mcp_gateway.py`

**新增功能**：
1. **缓存机制**：在`__init__`中添加 `_news_cache: dict[str, set[str]]`
2. **去重方法**：`_deduplicate_news()` - 基于标题hash去重
3. **智能重置**：当所有结果都被过滤时，清空缓存重新开始

**实现逻辑**：
```python
# 1. 计算标题hash
title_hash = hashlib.md5(title.encode()).hexdigest()

# 2. 检查是否已返回过
if title_hash not in self._news_cache[topic]:
    filtered_results.append(news)
    self._news_cache[topic].add(title_hash)

# 3. 如果全部过滤，清空缓存重新开始
if not filtered_results:
    self._news_cache[topic].clear()
    filtered_results = original_results[:3]
```

**效果**：
- ✅ 同一query连续搜索不会返回重复新闻
- ✅ 用户说"换一批"时返回不同内容
- ✅ 缓存满时自动清空，避免永远无结果

## 架构设计

### 新闻处理流程（优化后）
```

    ↓
_news() 检测类别
    ├─ 财经新闻 → news_provider
    │       ↓
    │   _deduplicate_news() ← 新增
    │       ↓
    │   返回精炼新闻
    │
    └─ 通用新闻 → baidu_web_search_provider
            ↓

            ↓
        _deduplicate_news() ← 新增
```
