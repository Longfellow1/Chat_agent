# 项目总结

## 项目概述

这是一个**行程规划助手 Demo**，用于演示如何通过 **Langchain 1.0** 框架集成 **MCP (Model Context Protocol)** 服务，实现 LLM 调用外部工具的能力。

## 核心目标

✅ **掌握 Langchain 1.0 框架的使用**  
✅ **学习 MCP 协议集成**  
✅ **理解 LLM 如何调用工具**  
✅ **快速验证产品原型**

## 技术实现

### 1. 后端架构

- **Node.js + Express**: 提供 RESTful API
- **Langchain 1.0**: 创建智能 Agent，协调 LLM 和工具
- **MCP 客户端**: 连接高德地图和小红书 MCP 服务
- **OpenAI API**: 提供 LLM 能力（GPT-4o-mini）

### 2. 前端架构

- **React + Vite**: 现代化前端框架
- **简洁 UI**: 用户友好的行程规划界面

### 3. 工作流程

```
用户输入 → 前端表单 → HTTP API → Langchain Agent
                                              ↓
                                    分析需求并选择工具
                                              ↓
                                    ┌─────────┴─────────┐
                                    ↓                   ↓
                              高德地图 MCP         小红书 MCP
                            (地点、路线)        (攻略、推荐)
                                    ↓                   ↓
                                    └─────────┬─────────┘
                                              ↓
                                       生成行程规划
                                              ↓
                                        返回前端展示
```

## 关键代码位置

### 1. Agent 创建 (`server/agent/planner.js`)

这是核心文件，展示了如何：
- 初始化 Langchain Agent
- 集成 MCP 工具
- 配置系统提示词
- 处理工具调用

```javascript
const agent = createAgent({

  systemMessage: '...'

```

### 2. MCP 客户端配置 (`server/mcp/`)

- `amap.js`: 高德地图 MCP 客户端
- `xhs.js`: 小红书 MCP 客户端

展示了如何：
- 使用 `MultiServerMCPClient` 连接 MCP 服务器
- 配置 HTTP/SSE 或 stdio transport
- 获取可用工具列表

### 3. API 端点 (`server/index.js`)

- `POST /api/plan`: 接收用户查询，调用 Agent，返回结果

### 4. 前端界面 (`client/src/App.jsx`)

- 用户输入表单
- 结果展示
- 加载状态处理

## 学习要点

### 1. Langchain Agent 的工作方式

Agent 会自动：
- 分析用户查询
- 选择合适的工具
- 调用工具并获取结果
- 整合信息生成回复

### 2. MCP 工具集成

MCP (Model Context Protocol) 提供标准化的工具接口：
- 工具发现
- 参数传递
- 结果返回
- 错误处理

### 3. 工具调用流程

1. **用户查询**: "想去北京玩3天，喜欢美食"
2. **Agent 分析**: 需要地点信息、路线规划、美食推荐
3. **工具选择**: 
   - 高德地图: 搜索北京景点、餐厅
   - 小红书: 搜索美食攻略
4. **工具调用**: Agent 自动调用选定的工具
5. **结果整合**: Agent 将工具结果整合成行程规划
6. **返回用户**: 展示完整的行程建议

## 快速开始

### 1. 安装依赖

```bash
npm run install:all
```

### 2. 配置环境变量

复制 `env.example` 为 `.env`，填入：
- `OPENAI_API_KEY`: OpenAI API 密钥（必需）
- `AMAP_API_KEY`, `AMAP_MCP_SERVER_URL`: 高德地图配置（可选）
- `XHS_API_KEY`, `XHS_MCP_SERVER_URL`: 小红书配置（可选）

### 3. 启动服务

```bash
npm run dev
```

### 4. 访问应用

打开 http://localhost:5173

## 注意事项

1. **API 密钥安全**: 不要将 `.env` 文件提交到版本控制
2. **MCP 服务配置**: 需要根据实际的高德地图和小红书 MCP 服务文档调整配置
3. **成本控制**: OpenAI API 调用会产生费用，注意控制使用量
4. **错误处理**: 当前版本为演示用途，生产环境需要更完善的错误处理
