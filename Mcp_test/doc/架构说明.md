## 技术架构

### 后端架构

```
用户请求 (HTTP POST /api/plan)
    ↓
Express 服务器 (server/index.js)
    ↓
Langchain Agent (server/agent/planner.js)
    ↓
    ├─→ LLM (OpenAI GPT-4o-mini)
    │
    └─→ MCP 工具
        ├─→ 高德地图 MCP (server/mcp/amap.js)
        │   └─→ MultiServerMCPClient
        │       └─→ 高德地图 MCP 服务器
        │
        └─→ 小红书 MCP (server/mcp/xhs.js)
            └─→ MultiServerMCPClient
                └─→ 小红书 MCP 服务器
```

### 核心组件说明

#### 1. Langchain Agent (`server/agent/planner.js`)

- **职责**: 协调 LLM 和 MCP 工具，处理用户查询并生成行程规划
- **技术**: Langchain 1.0 `createAgent`
- **流程**:
  1. 接收用户查询
  2. LLM 分析需求
  3. 自动选择合适的 MCP 工具
  4. 调用工具获取数据（地点、攻略等）
  5. 整合信息生成行程规划

#### 2. MCP 客户端 (`server/mcp/`)

- **高德地图 MCP** (`amap.js`):
  - 连接高德地图 MCP 服务器
  - 提供地点搜索、路线规划等工具
  - 支持 HTTP/SSE 和 stdio 两种传输方式

- **小红书 MCP** (`xhs.js`):
  - 连接小红书 MCP 服务器
  - 提供攻略搜索、内容推荐等工具
  - 支持 HTTP/SSE 和 stdio 两种传输方式

#### 3. Express 服务器 (`server/index.js`)

- **API 端点**:
  - `GET /health`: 健康检查
  - `POST /api/plan`: 行程规划接口

### 前端架构

```
React 应用 (client/src/App.jsx)
    ↓
用户输入表单
    ↓
HTTP POST /api/plan
    ↓
显示结果（行程规划）
```

### 数据流

1. **用户输入** → 前端收集行程需求
2. **API 请求** → POST `/api/plan` 发送到后端
3. **Agent 处理** → Langchain Agent 接收查询
4. **工具调用** → Agent 根据需求调用 MCP 工具
   - 高德地图: 搜索地点、规划路线
   - 小红书: 搜索攻略、推荐内容
5. **结果生成** → Agent 整合信息，生成行程规划
6. **返回前端** → JSON 格式返回结果
7. **展示结果** → 前端渲染行程规划

## MCP 集成方式

### 方式 1: HTTP/SSE Transport

适用于远程 MCP 服务器：

```javascript
{
  transport: 'sse',
  url: 'https://mcp.amap.com/v1',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
}
```

### 方式 2: STDIO Transport

适用于本地 MCP 服务器：

```javascript
{
  transport: 'stdio',
  command: 'npx',
  args: ['-y', '@amap/mcp-server']
}
```

## 关键学习点

### 1. Langchain Agent 创建

```javascript
const agent = createAgent({
  llm: model,
  tools: allTools,
  systemMessage: '...' // 指导 Agent 如何使用工具
});
```

### 2. MCP 工具集成

```javascript
const client = new MultiServerMCPClient({
  mcpServers: {
    serverName: { /* config */ }
  }
});
const tools = await client.getTools();
```

### 3. Agent 工具调用

Agent 会自动：
- 分析用户查询
- 选择合适的工具
- 传递正确的参数
- 处理工具返回结果
