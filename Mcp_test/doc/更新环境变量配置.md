# 🔧 更新 .env 配置

## 当前问题

你的 `.env` 文件中的高德地图 MCP URL 配置不正确：
```bash
AMAP_MCP_SERVER_URL=https://mcp.amap.com/v1  # ❌ 错误的路径
```

虽然代码已经修复可以自动处理，但**强烈建议使用 Node.js I/O 模式**（官方推荐）。

## ✅ 推荐方案：使用 Node.js I/O 模式

编辑 `.env` 文件，修改高德地图配置部分：

**删除或注释掉这行：**
```bash
# AMAP_MCP_SERVER_URL=https://mcp.amap.com/v1
```

**添加以下配置：**
```bash
AMAP_MCP_COMMAND=npx
AMAP_MCP_ARGS=-y,@amap/amap-maps-mcp-server
AMAP_API_KEY=REDACTED
```

**完整的 .env 文件高德部分应该是：**
```bash
# ========== 高德地图 MCP 配置（推荐使用 Node.js I/O 模式）==========
AMAP_MCP_COMMAND=npx
AMAP_MCP_ARGS=-y,@amap/amap-maps-mcp-server
AMAP_API_KEY=REDACTED
```

## 🔄 或者：继续使用 SSE 方式（已自动修复）

如果你想继续使用 SSE 方式，代码已经自动修复了 URL 路径：

```bash
AMAP_MCP_SERVER_URL=https://mcp.amap.com/api/v1/sse?key=REDACTED
AMAP_API_KEY=REDACTED
```

或者更简单（代码会自动添加 key）：
```bash
AMAP_MCP_SERVER_URL=https://mcp.amap.com/api/v1/sse
AMAP_API_KEY=REDACTED
```

**注意：** 即使使用旧的 `/v1` URL，代码也会自动修复为 `/api/v1/sse`。

## 📋 修改步骤

1. **打开 `.env` 文件**
2. **找到高德地图配置部分**
3. **按推荐方案修改**（使用 Node.js I/O 模式）
4. **保存文件**
5. **重启后端服务器**

## 🚀 重启后应该看到

使用 Node.js I/O 模式时，应该看到：
```
✅ 高德地图 MCP 客户端已连接，可用工具数: X
  - amap_xxx: 工具描述
  ...
```

使用 SSE 方式（自动修复后），应该看到：
```
ℹ️  自动修复高德地图 MCP URL: https://mcp.amap.com/v1 -> https://mcp.amap.com/api/v1/sse
✅ 高德地图 MCP 客户端已连接，可用工具数: X
```

## ❓ 为什么推荐 Node.js I/O 模式？

1. ✅ **官方推荐**：高德地图官方文档推荐使用这种方式
2. ✅ **更稳定**：不需要处理 URL 和网络连接问题
3. ✅ **自动更新**：使用 `npx -y` 会自动使用最新版本
4. ✅ **更简单**：不需要配置复杂的 URL 和参数
