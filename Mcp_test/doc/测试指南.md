# 测试指南

## ✅ 环境已准备就绪

项目依赖已全部安装完成，可以直接开始测试！

## 🚀 快速测试步骤

### 1. 配置 API Key（必需）

编辑 `.env` 文件，至少需要配置：

```bash
OPENAI_API_KEY=sk-your-actual-api-key-here
```

**获取 OpenAI API Key:**
- 访问 https://platform.openai.com/api-keys
- 登录或注册账号
- 创建新的 API Key
- 复制并粘贴到 `.env` 文件中

### 2. 启动项目

```bash
# 方式1: 同时启动前后端（推荐）
npm run dev

# 方式2: 分别启动
# 终端1: 后端
npm run server

# 终端2: 前端  
npm run client
```

### 3. 访问应用

打开浏览器访问：**http://localhost:5173**

### 4. 测试功能

1. **健康检查**
   - 访问 http://localhost:3001/health
   - 应该看到 `{"status":"ok","message":"Trip Planner API is running"}`

2. **行程规划测试**
   - 在表单中输入：`想去北京玩3天，喜欢历史文化和美食`
   - 点击"开始规划行程"
   - 等待 AI 生成行程规划

## 📝 关于 MCP 服务

### 当前状态

项目已配置好 MCP 客户端代码，但**高德地图和小红书 MCP 服务是可选的**：

- ✅ **不配置 MCP 也可以运行** - Agent 会以仅 LLM 模式运行，仍能给出行程建议
- ⚠️ **配置 MCP 可以获得更准确的数据** - 但需要实际的 MCP 服务地址

### 如果需要配置 MCP 服务

#### 高德地图 MCP

在 `.env` 中添加：

```bash
AMAP_API_KEY=your_amap_api_key
AMAP_MCP_SERVER_URL=https://mcp.amap.com/v1
```

或使用 stdio transport（如果高德提供本地服务器）：

```bash
AMAP_MCP_COMMAND=npx
AMAP_MCP_ARGS=-y,@amap/mcp-server
```

#### 小红书 MCP

在 `.env` 中添加：

```bash
XHS_API_KEY=your_xiaohongshu_api_key
XHS_MCP_SERVER_URL=https://api.xiaohongshu.com/mcp
```

或使用 stdio transport（如果使用 Python MCP 服务器）：

```bash
XHS_MCP_COMMAND=python
XHS_MCP_ARGS=-m,xiaohongshu_mcp
```

## 🔍 故障排查

### 问题1: 服务器启动失败

**检查:**
- Node.js 版本是否 >= 18
- `.env` 文件是否存在
- `OPENAI_API_KEY` 是否已配置

**查看日志:**
```bash
npm run server
```

### 问题2: Agent 初始化失败

**可能原因:**
- `OPENAI_API_KEY` 无效
- 网络连接问题

**解决方案:**
- 检查 API Key 是否正确
- 检查网络连接
- 查看控制台错误信息

### 问题3: 前端无法连接后端

**检查:**
- 后端是否运行在 http://localhost:3001
- 查看浏览器控制台错误
- 检查 `vite.config.js` 中的代理配置

### 问题4: MCP 工具未加载

**这是正常的！** 如果未配置 MCP 服务，会看到警告信息：
```
⚠️  高德地图 MCP 客户端连接失败，将在演示模式下运行
⚠️  小红书 MCP 客户端连接失败，将在演示模式下运行
```

Agent 仍会工作，只是不使用 MCP 工具。
