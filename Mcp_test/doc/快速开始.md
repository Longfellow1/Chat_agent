# 快速开始指南

## 前置要求

- Node.js 18+ 
- npm 或 yarn
- OpenAI API Key（或其他兼容 Langchain 的 LLM API Key）

## 安装步骤

### 1. 安装依赖

```bash
# 安装根目录和客户端依赖
npm run install:all
```

或者分别安装：

```bash
# 根目录依赖
npm install

# 客户端依赖
cd client
npm install
cd ..
```

### 2. 配置环境变量

复制环境变量示例文件：

```bash
cp env.example .env
```

编辑 `.env` 文件，填入必要的配置：

```env
# 必需: OpenAI API Key
OPENAI_API_KEY=sk-your-openai-api-key-here

# 可选: 高德地图 MCP 配置（如果可用）
AMAP_API_KEY=your-amap-api-key
AMAP_MCP_SERVER_URL=https://mcp.amap.com/v1

# 可选: 小红书 MCP 配置（如果可用）
XHS_API_KEY=your-xiaohongshu-api-key
XHS_MCP_SERVER_URL=https://api.xiaohongshu.com/mcp

# 服务器端口（默认 3001）
PORT=3001
```

### 3. 启动开发服务器

```bash
# 同时启动后端和前端（推荐）
npm run dev
```

或者分别启动：

```bash
# 终端1: 启动后端
npm run server

# 终端2: 启动前端
npm run client
```

### 4. 访问应用

打开浏览器访问：http://localhost:5173

## 使用说明

1. 在表单中输入您的行程需求
2. 可选：填写目的地、天数、偏好等信息
3. 点击"开始规划行程"
4. 等待 AI 调用 MCP 工具生成行程规划
5. 查看生成的行程建议

## 演示模式

如果 MCP 服务未配置或连接失败，Agent 仍会以仅 LLM 模式运行，可以根据常识给出行程建议。

## 常见问题

### 1. MCP 工具连接失败

- 检查 MCP 服务的 URL 和认证信息是否正确
- 查看控制台日志了解具体错误信息
- Agent 仍可在没有 MCP 工具的情况下运行

### 2. OpenAI API 错误

- 确认 `OPENAI_API_KEY` 已正确设置
- 检查 API Key 是否有效且有足够的额度
- 可以尝试使用其他兼容的模型（修改 `OPENAI_MODEL` 环境变量）

### 3. 端口冲突

- 后端默认端口：3001
- 前端默认端口：5173
- 可在 `.env` 和 `vite.config.js` 中修改
