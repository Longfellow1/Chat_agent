# 行程规划助手 Demo

基于 Langchain 1.0 框架，通过 LLM 接入高德地图和小红书 MCP 的行程规划助手演示项目。

## 项目目标

- 掌握 Langchain 1.0 框架的使用
- 学习如何通过 MCP（Model Context Protocol）集成外部服务
- 实现一个可快速验证的行程规划产品 demo

## 技术栈

- **后端**: Node.js + Express + Langchain 1.0
- **前端**: React + Vite
- **MCP 服务**: 高德地图 MCP、小红书 MCP
- **LLM**: OpenAI (或其他兼容 Langchain 的模型)

## 快速开始

### 1. 安装依赖

```bash
npm run install:all
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env` 并填入配置：

```bash
cp .env.example .env
```

必需的环境变量：
- `OPENAI_API_KEY`: OpenAI API 密钥
- `AMAP_API_KEY`: 高德地图 API 密钥（如果高德 MCP 需要）
- `XIAOHONGSHU_API_KEY`: 小红书 API 密钥（如果小红书 MCP 需要）

### 3. 启动项目

```bash
npm run dev
```

这将同时启动后端服务器（端口 3001）和前端开发服务器（端口 5173）。

### 4. 访问应用

打开浏览器访问 http://localhost:5173

## 功能说明

### 核心流程

1. 用户在前端输入行程需求（如：想去北京玩3天，喜欢美食和景点）
2. 前端发送请求到后端 API
3. 后端使用 Langchain Agent 处理请求
4. Agent 根据需求调用相应的 MCP 工具：
   - 高德地图 MCP：获取地点信息、路线规划、天气等
   - 小红书 MCP：获取相关旅游攻略、美食推荐等
5. Agent 整合信息，生成行程规划
6. 返回结果给前端展示

### MCP 工具调用示例

Agent 会自动选择合适的工具：
- 搜索地点：调用高德地图 MCP 的搜索接口
- 获取攻略：调用小红书 MCP 的搜索接口
- 路线规划：调用高德地图 MCP 的路线规划接口

## 学习要点

- Langchain Agent 的创建和使用
- MCP 客户端的配置和连接
- 如何让 LLM 选择合适的工具
- 工具调用的错误处理
- 前端与后端的集成

## 注意事项

- 本项目为演示用途，MCP 服务器配置可能需要根据实际的高德地图和小红书 MCP 服务进行调整
- 请确保 API 密钥的安全性，不要提交到版本控制系统
- 生产环境需要添加更多的错误处理和用户认证
