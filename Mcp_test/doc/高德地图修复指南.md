# 高德地图 MCP 配置修复指南

## 🔍 问题分析

根据[官方文档](https://lbs.amap.com/api/mcp-server/gettingstarted)，当前配置有以下问题：

1. **URL 错误**：使用的是 `https://mcp.amap.com/v1`，但应该是 `https://mcp.amap.com/api/v1/sse?key=<your_key>`
2. **Agent 参数错误**：`createAgent` 的参数应该是 `model` 而不是 `llm`

## ✅ 已修复的问题

1. ✅ 修正了 Agent 创建时的参数（`llm` → `model`）
2. ✅ 更新了高德地图 MCP 配置逻辑，支持正确的 URL 格式
3. ✅ 添加了对 Node.js I/O 模式的支持（官方推荐）

## 📋 推荐的配置方式

### 方式 1: Node.js I/O 模式（推荐）⭐

这是官方推荐的方式，最简单可靠。

**在 `.env` 文件中配置：**
```bash
AMAP_MCP_COMMAND=npx
AMAP_MCP_ARGS=-y,@amap/amap-maps-mcp-server
AMAP_API_KEY=REDACTED
```

**优点：**
- 官方推荐方式
- 不需要配置 URL
- 自动处理连接和认证

### 方式 2: SSE 方式

如果必须使用 HTTP/SSE 方式，URL 格式必须正确。

**在 `.env` 文件中配置：**
```bash
AMAP_MCP_SERVER_URL=https://mcp.amap.com/api/v1/sse?key=REDACTED
AMAP_API_KEY=REDACTED
```

**注意：**
- URL 必须是 `/api/v1/sse`（不是 `/v1`）
- URL 中必须包含 `?key=<your_key>` 参数
- 或者代码会自动添加 key 参数

1. 更新 `.env` 文件配置（选择上述方式之一）
2. 重启后端服务器：
   ```bash
   npm run server
   ```
3. 查看日志，应该看到：
   ```
   ✅ 高德地图 MCP 客户端已连接，可用工具数: X
   ```

## ⚠️ 注意事项

- **Node.js 版本要求**：使用 Node.js I/O 模式需要 Node.js v22.14.0 或更高版本
- **网络要求**：需要能够访问 npm registry 来下载 `@amap/amap-maps-mcp-server` 包
- **首次使用**：使用 Node.js I/O 模式时，首次运行会自动下载包，可能需要一些时间
