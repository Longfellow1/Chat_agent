# 🚀 正确的启动步骤

## ⚠️ 问题：后端未运行

`Failed to fetch` 错误说明前端无法连接到后端。**需要先启动后端！**

---

## ✅ 解决方案：分两步启动

### 步骤 1: 启动后端（必须）

打开**第一个终端**，运行：

```bash
cd /Users/Harland/Documents/Mcp_test
npm run server
```

**等待看到这些信息，说明后端启动成功：**
```
🚀 启动服务器，初始化 Agent...
🚀 初始化行程规划 Agent...
📦 加载 MCP 工具...
✅ 行程规划 Agent 初始化完成
🚀 行程规划 API 服务器运行在 http://localhost:3001
```

**保持这个终端打开！** 不要关闭，让后端继续运行。

### 步骤 2: 启动前端

打开**第二个终端**（新开一个终端窗口），运行：

```bash
cd /Users/Harland/Documents/Mcp_test
npm run client
```

**等待看到这些信息，说明前端启动成功：**
```
VITE v7.x.x  ready in xxx ms
➜  Local:   http://localhost:5173/
```

### 步骤 3: 访问应用

打开浏览器访问：**http://localhost:5173**

现在应该可以正常使用了！

## 🔍 如果后端启动失败

### 检查 1: OpenAI API Key 配置

确认 `.env` 文件中有正确的 API Key：

```bash
cat .env | grep OPENAI_API_KEY
```

应该看到类似：
```
OPENAI_API_KEY=sk-...
```

如果不是，编辑 `.env` 文件填入正确的 API Key。

### 检查 2: 查看具体错误

后端启动时如果有错误，会显示在终端中。常见错误：

**错误：缺少 OPENAI_API_KEY**
```
❌ Agent 初始化失败：缺少 OPENAI_API_KEY 环境变量
```

**解决：** 检查 `.env` 文件配置

**错误：API Key 无效**
```
❌ Agent 初始化失败：API Key 无效
```

**解决：** 检查 API Key 是否正确，是否有效

## 💡 为什么需要两个终端？

1. **后端服务器需要持续运行** - 它监听请求，不能关闭
2. **前端开发服务器也需要运行** - 它提供前端页面
3. **分别启动便于查看日志** - 可以看到各自的错误信息

## ❓ 常见问题

**Q: 能不能用一个命令同时启动？**

A: 可以运行 `npm run dev`，但如果遇到问题，建议分开启动，更容易定位错误。

**Q: 后端启动后看到警告信息正常吗？**

A: 如果是 MCP 相关的警告（如"小红书 MCP 未配置"），这是正常的，不影响使用。

**Q: 如何停止服务器？**

A: 在对应的终端按 `Ctrl + C`
