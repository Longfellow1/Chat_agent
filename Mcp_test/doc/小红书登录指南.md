# 小红书 MCP 扫码接入说明

## 📌 重要说明：小红书 MCP 是可选的！

**你可以完全不配置小红书 MCP，项目也能正常运行！**

- ✅ 项目主要功能（行程规划）**不需要**小红书 MCP
- ✅ 高德地图 MCP 已经足够提供地点、路线等信息
- ✅ 小红书 MCP 只是用来获取旅游攻略、美食推荐等**额外信息**
- ✅ 不配置小红书 MCP，Agent 仍可以正常生成行程规划

---

## 🤔 什么是扫码接入？

小红书 MCP 使用的是**个人账号登录**方式，而不是 API Key。

**工作原理：**
1. 运行小红书 MCP 服务器（Python 程序）
2. 首次运行时，会生成一个二维码
3. 用手机小红书 App 扫描二维码
4. 确认登录后，登录状态会保存
5. 后续使用无需重复扫码

**为什么需要扫码？**
- 小红书没有公开的 API，需要登录个人账号来访问内容
- 通过扫码登录，MCP 服务器可以访问你的小红书账号数据
- 这样就可以搜索攻略、获取推荐等

## 🚫 如果你不想使用小红书 MCP（推荐）

**完全可以！** 只需要：

1. **不要配置** `.env` 文件中的小红书相关配置
2. **直接启动项目**，使用高德地图 MCP 即可
3. 项目会以仅 LLM + 高德地图模式运行，完全够用


**不需要**添加任何小红书相关的配置。

## ✅ 如果你确实想使用小红书 MCP（可选）

### 前置要求

1. **需要 Python 环境**
   ```bash
   python --version  # 检查是否安装 Python 3.7+
   ```

2. **安装小红书 MCP 包**
   ```bash
   pip install xiaohongshu-mcp
   ```
   或者根据 GitHub 项目说明安装：https://github.com/xpzouying/xiaohongshu-mcp

### 配置步骤

1. **在 `.env` 文件中添加配置**
   ```bash
   XHS_MCP_COMMAND=python
   XHS_MCP_ARGS=-m,xiaohongshu_mcp
   ```

2. **启动项目**
   ```bash
   npm run dev
   ```

3. **首次运行时的扫码流程**
   - 当 MCP 服务器首次启动时，会在终端或日志中显示二维码
   - 打开手机上的小红书 App
   - 使用 App 内的扫码功能扫描二维码
   - 在手机上确认登录
   - 登录成功后，状态会保存到本地
   - 后续使用无需重复扫码

### 扫码示例

当首次运行时，你可能会在终端看到类似这样的输出：
```
请使用小红书 App 扫描以下二维码登录：
█████████████████████████████
█████████████████████████████
█████████████████████████████
...
```

然后用手机小红书 App 扫描即可。
