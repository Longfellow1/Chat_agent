## 🔧 需要配置

### 必需配置

1. **安装依赖**
   ```bash
   npm run install:all
   ```

2. **配置环境变量**
   - 复制 `env.example` 为 `.env`
   - 至少配置 `OPENAI_API_KEY`

### 可选配置

3. **高德地图 MCP**（如果可用）
   - 配置 `AMAP_API_KEY`
   - 配置 `AMAP_MCP_SERVER_URL` 或 `AMAP_MCP_COMMAND`

4. **小红书 MCP**（如果可用）
   - 配置 `XHS_API_KEY`
   - 配置 `XHS_MCP_SERVER_URL` 或 `XHS_MCP_COMMAND`

## ⚠️ 已知问题和注意事项

### 1. MCP 服务配置

- 高德地图和小红书的 MCP 服务器 URL/命令需要根据实际情况调整
- 如果 MCP 服务未配置，Agent 会以仅 LLM 模式运行
- 参考实际的高德地图和小红书 MCP 文档进行配置

### 2. Langchain 包版本

- 当前使用 `langchain` 0.3.0
- 如果遇到导入问题，可能需要调整版本
- `createAgent` 应该从 `langchain` 导入

### 3. Node.js 版本

- 建议使用 Node.js 18+
- 确保支持 ES modules（`"type": "module"`）

## 🚀 测试步骤

1. **启动服务器**
   ```bash
   npm run server
   ```
   检查是否看到：
   - "🚀 启动服务器，初始化 Agent..."
   - Agent 初始化成功或警告信息
   - "🚀 行程规划 API 服务器运行在 http://localhost:3001"

2. **启动前端**
   ```bash
   npm run client
   ```
   或使用 `npm run dev` 同时启动

3. **测试 API**
   ```bash
   curl http://localhost:3001/health
   ```

4. **测试完整流程**
   - 打开 http://localhost:5173
   - 输入行程需求
   - 点击"开始规划行程"
   - 查看结果
