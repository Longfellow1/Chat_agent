# 🔧 端口占用问题解决方案

## 问题：端口 3001 已被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::3001
```

**原因：** 之前的服务器进程还在运行，占用了端口 3001。

---

## ✅ 解决方案

### 方法 1: 终止占用端口的进程（推荐）

```bash
# 查找占用端口 3001 的进程并终止
lsof -ti:3001 | xargs kill -9
```

或者一步完成：
```bash
kill -9 $(lsof -ti:3001)
```

### 方法 2: 终止所有相关 Node 进程

```bash
# 终止所有 server/index.js 相关进程
pkill -f "node.*server/index.js"
pkill -f "node --watch.*server"
```

### 方法 3: 修改端口（如果不想终止现有进程）

编辑 `.env` 文件，修改端口：
```bash
PORT=3002
```

然后更新 `client/vite.config.js` 中的代理端口：
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:3002',  // 改为新端口
    changeOrigin: true,
  },
}
```

---

## 🔍 检查端口占用

**查看哪个进程占用端口：**
```bash
lsof -i:3001
```

**查看所有 Node 进程：**
```bash
ps aux | grep node | grep -v grep
```

---

## 💡 预防措施

已更新代码，如果端口被占用会显示友好的错误提示。

**快速解决命令：**
```bash
# 一键清理端口占用
lsof -ti:3001 | xargs kill -9 && npm run server
```

---

## 📝 完整的启动流程

```bash
# 1. 清理端口（如果需要）
lsof -ti:3001 | xargs kill -9

# 2. 启动后端
npm run server

# 3. 新终端启动前端
npm run client
```
