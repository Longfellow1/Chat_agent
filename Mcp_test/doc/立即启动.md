# 🚀 立即启动项目

## ✅ 配置检查

你已经配置了：
- ✅ 高德地图 API Key: `REDACTED`
- ✅ 高德地图 MCP 服务器地址

**还需要配置（必需）：**
- ⚠️ OpenAI API Key（必需，否则无法启动）

---

## 📝 步骤 1: 配置 OpenAI API Key（如果还没配置）

编辑 `.env` 文件，将以下内容：
```bash
OPENAI_API_KEY=your_openai_api_key_here
```

替换为你的实际 API Key：
```bash
OPENAI_API_KEY=sk-your-actual-openai-api-key
```

**获取 OpenAI API Key:**
1. 访问 https://platform.openai.com/api-keys
2. 登录/注册账号
3. 创建新的 API Key
4. 复制并粘贴到 `.env` 文件中

---

## 🚀 步骤 2: 启动项目

### 方式一：同时启动前后端（推荐）

```bash
npm run dev
```

这将同时启动：
- 后端服务器: http://localhost:3001
- 前端应用: http://localhost:5173

### 方式二：分别启动（便于查看日志）

**终端 1 - 启动后端：**
```bash
npm run server
```

**终端 2 - 启动前端：**
```bash
npm run client
```

---

## ✅ 步骤 3: 验证启动成功

### 1. 查看后端日志

启动后端后，你应该看到类似这样的输出：

```
🚀 启动服务器，初始化 Agent...
🚀 初始化行程规划 Agent...
📦 加载 MCP 工具...
✅ 高德地图 MCP 客户端已连接，可用工具数: X
  - amap_search_place: 地点搜索
  - amap_route_plan: 路线规划
  ...
✅ 行程规划 Agent 初始化完成
🚀 行程规划 API 服务器运行在 http://localhost:3001
```

**如果看到这个，说明高德地图 MCP 连接成功！** ✅

**如果看到警告信息：**
```
⚠️  高德地图 MCP 客户端连接失败，将在演示模式下运行
```

可能的原因：
- MCP 服务器地址不正确（需要查看高德官方文档确认实际地址）
- 网络连接问题
- API Key 权限问题

### 2. 测试后端健康检查

打开浏览器或使用 curl：

```bash
curl http://localhost:3001/health
```

应该返回：
```json
{"status":"ok","message":"Trip Planner API is running"}
```

### 3. 访问前端应用

打开浏览器访问：**http://localhost:5173**

你应该看到行程规划助手的界面。

## 🧪 步骤 4: 测试高德地图 MCP

### 测试用例

在前端表单中输入：

```
想去北京玩3天，喜欢历史文化和美食
```

或者更详细的：

- **行程需求**: `想去北京旅游，参观历史景点和品尝美食`
- **目的地**: `北京`
- **天数**: `3`
- **偏好**: `历史、美食、文化`

### 预期结果

1. 点击"开始规划行程"后，会显示加载状态
2. Agent 会调用高德地图 MCP 工具：
   - 搜索北京的地点
   - 规划路线
   - 查找附近的餐厅和景点
3. 生成包含实际地点信息的行程规划

### 如何验证高德地图 MCP 是否工作

**方法 1: 查看后端日志**
- 在终端中查看详细的工具调用日志
- 如果看到 `amap_` 开头的工具被调用，说明 MCP 正常工作

**方法 2: 查看前端结果**
- 行程规划中应该包含具体的地址、路线等实际数据
- 而不是模糊的建议

## 🛠️ 故障排查

### 问题 1: 后端启动失败

**检查：**
- `.env` 文件中的 `OPENAI_API_KEY` 是否已配置
- 查看错误日志信息

### 问题 2: 高德地图 MCP 连接失败

**可能原因：**
1. MCP 服务器地址不正确
   - 当前配置: `https://mcp.amap.com/v1`
   - 需要查看高德官方文档确认实际地址
   
2. API Key 权限问题
   - 确认申请 Key 时选择了 "Web服务"
   - 检查 IP 白名单设置

3. 网络问题
   - 检查网络连接
   - 尝试直接访问 MCP 服务器地址

**解决方案：**
- 查看后端日志中的具体错误信息
- 参考高德地图 MCP 官方文档：https://lbs.amap.com/api/mcp-server/summary
- 即使 MCP 连接失败，Agent 仍可以运行（仅 LLM 模式）

### 问题 3: 前端无法连接后端

**检查：**
- 后端是否正常运行在 http://localhost:3001
- 查看浏览器控制台的错误信息
- 检查 `vite.config.js` 中的代理配置
