### 第二阶段：用户反馈修复（3个问题）

#### 问题1：重写提示词太精炼，没人味儿
**解决方案**：优化LLM重写prompt，增加口语化
- 改为"有趣的新闻编辑"角色
- 支持"据说"、"看来"、"竟然"等口语词汇
- 保留关键信息但生动有趣

**示例对比**：
```
原：一句话摘要，不超过50字。
新：今年春节档真是热闹，各种题材的电影都有，票房创新高，看来大家过年都爱去电影院。
```

#### 问题2：相关性排序不明确
**解决方案**：提高相关性阈值，过滤不相关结果
- 阈值从0.1提升到0.3
- 综合评分 = 相关性 * 0.7 + 可信度 * 0.3
- 最相关内容自动排在最前面

#### 问题3：重复搜索无去重
**解决方案**：实现会话级别的去重机制
- 添加`_news_cache`缓存（query → title_hash集合）
- 基于标题MD5 hash去重
- 所有结果过滤时自动清空缓存重新开始

**实现逻辑**：
```python
# 1. 计算标题hash
title_hash = hashlib.md5(title.encode()).hexdigest()

# 2. 检查是否已返回过
if title_hash not in self._news_cache[topic]:
    filtered_results.append(news)
    self._news_cache[topic].add(title_hash)

# 3. 全部过滤时清空缓存
if not filtered_results:
    self._news_cache[topic].clear()
```

## 架构设计

### 新闻处理流程
```
用户查询
    ↓
_news() 检测类别
    ├─ 财经新闻 → news_provider
    │       ↓
    │   _deduplicate_news() ← 去重
    │       ↓
    │   返回精炼新闻
    │
    └─ 通用新闻 → baidu_web_search_provider
            ↓
        相关性过滤（threshold=0.3）
            ↓
        _deduplicate_news() ← 去重
            ↓
        _apply_news_rewrite() ← 口语化
            ↓
        返回精炼新闻
```
