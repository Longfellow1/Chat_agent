# Parser.py 评审意见

## 问题1：_extract_city 的 invalid_keywords 太脆弱

```python
invalid_keywords = ["鸟", "巢", "便", "利", "店", "周", "边", "附", "近", "的", "超", "市场"]
```

**问题**：
- 单字过滤是定时炸弹："常州市"、"义乌市"完全合法，但如果query里恰好有这些单字就会误杀
- "超市场"这种切法说明在用字符串包含做逻辑判断，这不是正则该干的活

**根本问题**：
- 用黑名单防止"超市"被误识别为城市
- 正确做法：**whitelist城市，regex只作为最后fallback**，已知城市表要足够完整

## 问题2：_extract_anchor_poi 的剥离顺序有竞争条件

```python
# Remove brands → Remove categories → Remove keywords → 剩余当anchor
```

**"剥洋葱"策略的脆弱性**：

✅ "春熙路附近的海底捞" → 剥掉海底捞、附近 → 剩"春熙路"  
❌ "海底捞春熙路店" → 剥掉海底捞 → 剩"春熙路店" → "店"没被移除  
⚠️ "五四广场附近的咖啡" → location_pattern先匹配"广场" ✅ 但"五四"这种数字+汉字混合，后面`re.sub(r"\d+", "", q)`会把"五"留下变成残缺

**问题**：
- 依赖query结构规整
- 对倒装结构失效
- 数字地名处理矛盾

## 问题3：district提取会吃掉anchor

```python
# _extract_district里：
q = re.sub(r"[\u4e00-\u9fa5]{2,4}市", "", q)
match = re.search(r"([\u4e00-\u9fa5]{2,6})区", q)
```

**场景分析**：
- "朝阳区三里屯的星巴克" → district="朝阳区"，anchor提取时remove district，"三里屯"能留下 ✅
- "朝阳区的一个安静的咖啡厅" → district吃掉朝阳区之后，anchor那里什么都没剩，这时候应该用district本身作为anchor的fallback，但代码没有这个逻辑 ❌

## 正确的设计思路（参考workflow node 100138）

### 1. 城市提取：完整whitelist + 最长匹配

```python
all_cities = {
    "北京", "上海", "天津", "重庆", ...  # 完整的~300个地级市
}

# 按长度降序排列，避免"中山"覆盖"厦门中山路"
sorted_cities = sorted(all_cities, key=len, reverse=True)
for city in sorted_cities:
    if city in user_input:
        hit_city = city
        break
```

### 2. 数据清洗：解决city/district重复

```python
# 如果提取到了城市，且区县里也包含了这个城市名
# 比如 city="广州", district="广州白云区" -> 把"广州"删掉，剩下"白云区"
if hit_city and hit_district and hit_district.startswith(hit_city):
    hit_district = hit_district.replace(hit_city, "", 1)
```

### 3. Anchor提取：正向匹配，不依赖剥离

**关键原则**：
- 不要用"剥洋葱"策略（Remove brands → Remove categories → 剩余当anchor）
- 用**正向pattern匹配**：
  1. 先匹配known landmarks字典
  2. 再匹配"XX(附近|周边|旁边)"pattern
  3. 最后匹配POI suffix pattern（路|街|广场|站）
- 匹配成功后**直接返回**，不进入数字清理逻辑

### 4. 解析顺序

```python
# 1. 提取city
city = _extract_city(query)

# 2. 提取district
district = _extract_district(query)

# 3. 数据清洗
if city and district and district.startswith(city):
    district = district.replace(city, "", 1)

# 4. 提取anchor（在移除city/district后的query上操作）
anchor = _extract_anchor_poi(query)

# 5. 提取category（在移除anchor后的query上操作）
query_clean = query.replace(anchor, "")
category = parse_category(query_clean)
```
