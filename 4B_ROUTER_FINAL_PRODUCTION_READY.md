## 修复内容

### Bug 1：规则层「城市+旅游」映射错误 🔴

**问题**：
```python
# 修复前
"广州旅游" → 匹配「广州」(location) + "旅游"→"景点"(category)
         → find_nearby ❌ 错误
```

**根因**：规则2（位置+类别）优先级高于旅游意图，导致「城市+旅游」被错误映射到 find_nearby

**修复**：
```python
# 修复后：新增规则0，优先级最高
TRIP_KEYWORDS = ["旅游", "旅行", "出去玩", "出去走走", "散散心", 
                 "出去转转", "避暑", "避寒", "度假", "度个假"]

# 规则0：旅游意图关键词 = plan_trip（优先级最高）
if any(kw in query for kw in TRIP_KEYWORDS):
    return plan_trip
```

**效果**：
- id=30 "广州旅游" → plan_trip ✅（修复前 find_nearby ❌）
- id=86 "出去转转" → plan_trip ✅（修复前 web_search ❌）
- id=87 "散散心" → plan_trip ✅（修复前 web_search ❌）
- id=89 "避暑" → plan_trip ✅（修复前 web_search ❌）
- id=90 "避寒" → plan_trip ✅（修复前 get_weather ❌）

### 部署架构

```
用户 query
    ↓
规则层（旅游词、城市+类别、天气）
    ├─ 命中 → 返回（0ms，100% 准确）
    └─ 未命中 ↓
4B LLM（处理复杂 query）
    ├─ 置信度 ≥ 0.7 → 返回
    └─ 置信度 < 0.7 → 澄清
```

### 置信度策略（需谨慎）

**问题**：高置信度错误（confidence=0.85，答错）是最危险的

**当前数据**：
- 置信度 0.85 的 case 中，仍有部分答错
- 这些错误不会触发澄清流程
- 需要监控高置信度错误率

**建议策略**：

| 置信度 | 动作 | 说明 |
|--------|------|------|
| 0.90（规则层） | 直接执行 | 100% 准确 |
| 0.85（LLM 高置信） | 直接执行 | 需监控错误率 |
| 0.65-0.84 | 直接执行 | 中等置信 |
| < 0.65 | 触发澄清 | 低置信 |

**监控指标**：
- 高置信度错误率（0.85）
- 如果错误率 > 5%，考虑提高阈值到 0.9
