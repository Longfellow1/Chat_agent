# 通用新闻工具第一轮优化完成

## 问题分析

用户反馈：搜索"找找娱乐新闻"时，返回结果包含URL、日期等元数据，且LLM重写没有生效。

**根本原因**：
- 娱乐新闻不是财经新闻，走的是 `get_news_general_chain`
- 使用 `baidu_web_search_provider`，返回原始搜索结果
- 没有应用content_rewriter进行LLM重写

## 修改内容

### 1. Provider层优化 (`baidu_web_search_provider.py`)
**改动**：调整返回格式，移除URL和日期

```
原格式：
1. 标题 [官方] | http://url | 摘要

新格式：
1. 标题
摘要（100字以内）
```

**关键改进**：
- 移除URL、日期、信任标记等元数据
- 摘要截断到100字以内
- 格式简洁：序号. 标题\n摘要

### 2. Gateway层优化 (`mcp_gateway.py`)
**改动**：在_news方法中应用content_rewriter进行LLM重写

**新增方法**：`_apply_news_rewrite()`
- 调用LLM对新闻进行重写
- 生成一句话摘要（50字以内）
- 失败时降级到原始输出

**流程改进**：
```
get_news_general_chain.execute()
    ↓
_apply_news_rewrite()  ← 新增
    ↓
返回精炼新闻
```

## 架构设计

### 新闻处理流程
```
用户查询
    ↓
_news() 检测类别
    ├─ 财经新闻 → news_provider (Sina News)
    └─ 通用新闻 → baidu_web_search_provider
                    ↓
                _apply_news_rewrite()
                    ↓
                返回精炼新闻
```

### 两条链路的区别

| 链路 | Provider | 特点 | 输出 |
|------|----------|------|------|
| 财经 | news_provider | 专用财经API | 精炼摘要 |
| 通用 | baidu_web_search | 通用搜索 | 精炼摘要 |
